📋 ШАГ 1: УСТАНОВКА ЗАВИСИМОСТЕЙ

   pip install web3==6.15.1 eth-account==0.10.0 requests==2.31.0

   ИЛИ из файла:
   pip install -r requirements.txt


🔑 ШАГ 2: НАСТРОЙКА token_collector.py

   Откройте файл и настройте следующие параметры (строки 17-23):

   ┌─────────────────────────────────────────────────────────────────────┐
   │ RECIPIENT_ADDRESS = "0x..."          # Адрес получателя токенов     │
   │ RECIPIENT_ADDRESS_KEY = "0x..."     # Приватный ключ получателя    │
   │ DONOR_FILE = "DONOR.txt"            # Файл с ключами кошельков     │
   │ AUTO_REFUEL_GAS = True              # Автопополнение газа          │
   │ MIN_GAS_REFUEL_AMOUNT = 0.002       # Сумма пополнения (ETH)       │
   │ MIN_BALANCE_USD = 0.05              # Мин. баланс для обработки    │
   │ MAX_PARALLEL_WALLETS = 10           # Кол-во потоков               │
   └─────────────────────────────────────────────────────────────────────┘

   ⚠️ ВАЖНО:
   • RECIPIENT_ADDRESS - адрес куда собираются все токены
   • RECIPIENT_ADDRESS_KEY - нужен для автопополнения газа (необязательно)
   • Если не хотите автопополнение: AUTO_REFUEL_GAS = False


📝 ШАГ 3: ПОДГОТОВКА DONOR.txt

   Создайте файл DONOR.txt и добавьте приватные ключи (по одному на строку):

   ┌─────────────────────────────────────────────────────────────────────┐
   │ 0xabc123def456...приватный_ключ_1                                   │
   │ 0x789ghi012jkl...приватный_ключ_2                                   │
   │ 0x345mno678pqr...приватный_ключ_3                                   │
   │ # Комментарии начинаются с #                                        │
   └─────────────────────────────────────────────────────────────────────┘


🚀 ШАГ 4: ЗАПУСК

   python token_collector.py

   Скрипт автоматически:
   1. Загрузит приватные ключи из DONOR.txt
   2. Для каждого кошелька в каждой сети:
      • Проверит балансы LayerBank (если Scroll)
      • Проверит ликвидность SyncSwap (если zkSync ERA)
      • Проверит все ERC-20 токены
      • Соберет нативные токены
   3. Отправит все на RECIPIENT_ADDRESS


═══════════════════════════════════════════════════════════════════════════
                        АВТОПОПОЛНЕНИЕ ГАЗА
═══════════════════════════════════════════════════════════════════════════

🔋 КАК ЭТО РАБОТАЕТ:

   1. Перед каждой операцией скрипт проверяет баланс газа
   2. Если газа недостаточно:
      → Автоматически отправляет MIN_GAS_REFUEL_AMOUNT (0.002 ETH)
      → С кошелька RECIPIENT_ADDRESS
      → На донорский кошелек
   3. После пополнения продолжает операцию
   4. В конце возвращает неиспользованные остатки


📊 ПОРЯДОК ОПЕРАЦИЙ:

   Для каждого кошелька:
   
   1️⃣ LayerBank (Scroll)
      └─ Проверка газа → Автопополнение → Вывод депозитов
   
   2️⃣ SyncSwap (zkSync ERA)
      └─ Проверка газа → Автопополнение → Вывод ликвидности
   
   3️⃣ ERC-20 токены (все сети)
      └─ Для КАЖДОГО токена:
         Проверка газа → Автопополнение → Перевод
   
   4️⃣ Нативный токен
      └─ Отправка всех остатков на RECIPIENT_ADDRESS
   
   5️⃣ Возврат газа (если были автопополнения)
      └─ Возврат неиспользованных средств


⚙️ НАСТРОЙКИ АВТОПОПОЛНЕНИЯ:

   AUTO_REFUEL_GAS = True / False
   • True  - автопополнение включено
   • False - автопополнение выключено (работает как раньше)

   MIN_GAS_REFUEL_AMOUNT = 0.002
   • Сумма пополнения в нативном токене (ETH/BNB/POL/и т.д.)
   • Рекомендуется: 0.002-0.005 для большинства сетей
   • Для Ethereum можно увеличить до 0.01

   RECIPIENT_ADDRESS_KEY = "0x..."
   • Приватный ключ от RECIPIENT_ADDRESS
   • Нужен только если AUTO_REFUEL_GAS = True
   • ОБЯЗАТЕЛЬНО должен соответствовать RECIPIENT_ADDRESS


🔒 БЕЗОПАСНОСТЬ:

   ✅ Перед пополнением проверяется баланс RECIPIENT_ADDRESS
   ✅ Минимальный порог: MIN_GAS_REFUEL_AMOUNT × 2
   ✅ Неиспользованный газ возвращается автоматически
   ✅ Все операции логируются с tx_hash


📝 ПРИМЕРЫ ЛОГОВ:

                        НАСТРОЙКИ И ПАРАМЕТРЫ
═══════════════════════════════════════════════════════════════════════════

📝 ОСНОВНЫЕ НАСТРОЙКИ (строки 17-30):

   RECIPIENT_ADDRESS = "0x..."
     • Адрес получателя всех собранных токенов
     • Обязательный параметр

   RECIPIENT_ADDRESS_KEY = ""
     • Приватный ключ от RECIPIENT_ADDRESS
     • Нужен только для автопополнения газа
     • Если пустой - автопополнение недоступно

   DONOR_FILE = "DONOR.txt"
     • Файл с приватными ключами донорских кошельков
     • По одному ключу на строку

   GAS_PRICE_MULTIPLIER = 1.2
     • Множитель для gas price (20% запас)
     • Увеличивает шанс быстрого подтверждения

   MIN_BALANCE_USD = 0.05
     • Минимальный баланс токена для обработки ($0.05)
     • Токены с балансом меньше этого пропускаются

   AUTO_REFUEL_GAS = True
     • Включение автопополнения газа
     • True = включено, False = выключено

   MIN_GAS_REFUEL_AMOUNT = 0.002
     • Сумма пополнения газа в нативном токене
     • Рекомендуется: 0.002-0.005 ETH


⚡ НАСТРОЙКИ МНОГОПОТОЧНОСТИ (строки 29-30):

   MAX_PARALLEL_WALLETS = 10
     • Количество кошельков для одновременной обработки
     • Рекомендуется: 5-10 для стабильности
     • Больше = быстрее, но выше нагрузка на RPC

   PAUSE_BETWEEN_WALLETS = 2
     • Пауза между обработкой кошельков (секунды)
     • Предотвращает rate limiting от RPC

                      ЧАСТО ЗАДАВАЕМЫЕ ВОПРОСЫ
═══════════════════════════════════════════════════════════════════════════

❓ Нужен ли RECIPIENT_ADDRESS_KEY обязательно?

   НЕТ. Если вы не планируете использовать автопополнение газа:
   • Оставьте RECIPIENT_ADDRESS_KEY = ""
   • Установите AUTO_REFUEL_GAS = False
   • Убедитесь, что на донорских кошельках есть газ для транзакций


❓ Что делать если на донорских кошельках нет газа?

   ВАРИАНТ 1: Вручную пополните газ перед запуском
   ВАРИАНТ 2: Используйте автопополнение:
   • Заполните RECIPIENT_ADDRESS_KEY
   • Установите AUTO_REFUEL_GAS = True
   • Держите баланс на RECIPIENT_ADDRESS (~0.01-0.05 ETH на сеть)


❓ Сколько стоит автопополнение газа?

   • Комиссия за транзакцию пополнения: $0.50-5 (зависит от сети)
   • zkSync ERA: ~$0.10-0.30
   • Scroll: ~$0.20-0.50
   • Ethereum: ~$2-5
   • Остальные сети: ~$0.50-1


❓ Можно ли обрабатывать только определенные сети?

   ДА. Отредактируйте словарь NETWORKS (строка 67):
   • Удалите ненужные сети
   • ИЛИ закомментируйте их с помощью #


❓ Как добавить новые токены?

   Найдите нужную сеть в NETWORKS и добавьте токен:
   
   {"symbol": "NEW_TOKEN", "address": "0x..."}

   Убедитесь что:
   • Адрес правильный для этой сети
   • Токен имеет стандартный ERC-20 интерфейс


❓ Что делать если транзакция не проходит?

   1. Проверьте баланс газа на кошельке
   2. Увеличьте GAS_PRICE_MULTIPLIER (например, до 1.5)
   3. Для zkSync ERA может потребоваться больше времени
   4. Проверьте transaction hash в block explorer


❓ Сколько времени занимает обработка?

   Зависит от:
   • Количества кошельков
   • Количества токенов на каждом кошельке
   • Настройки MAX_PARALLEL_WALLETS
   • Скорости сети

   Примерно:
   • 1 кошелек с 5 токенами: ~2-5 минут
   • 10 кошельков параллельно: ~10-20 минут
   • 100 кошельков (10 параллельно): ~1-2 часа


❓ Безопасно ли использовать многопоточность?

   ДА. Скрипт использует:
   • threading.Lock для безопасного вывода
   • Отдельные Web3 инстансы для каждого потока
   • Паузы между операциями

                        РЕШЕНИЕ ПРОБЛЕМ
═══════════════════════════════════════════════════════════════════════════

🔴 ОШИБКА: "Не удалось подключиться к [сеть]"

   ПРИЧИНА: Проблемы с RPC
   РЕШЕНИЕ:
   • Проверьте интернет-соединение
   • RPC может быть временно недоступен
   • Ankr API KEY может быть недействителен


🔴 ОШИБКА: "Недостаточно средств у получателя"

   ПРИЧИНА: На RECIPIENT_ADDRESS нет газа для автопополнения
   РЕШЕНИЕ:
   • Пополните RECIPIENT_ADDRESS (минимум 0.01 ETH)
   • ИЛИ отключите автопополнение: AUTO_REFUEL_GAS = False


🔴 ОШИБКА: "Ошибка пополнения gas"

   ПРИЧИНА: Неправильный RECIPIENT_ADDRESS_KEY
   РЕШЕНИЕ:
   • Проверьте правильность приватного ключа
   • Убедитесь что ключ соответствует RECIPIENT_ADDRESS:
     python -c "from web3 import Web3; print(Web3().eth.account.from_key('ВАШ_КЛЮЧ').address)"


🔴 ПРЕДУПРЕЖДЕНИЕ: "Баланс меньше $0.05, пропускаем"

   ПРИЧИНА: Баланс токена слишком мал
   РЕШЕНИЕ:
   • Это нормально, токены с малым балансом игнорируются
   • Можно уменьшить MIN_BALANCE_USD если нужно


🔴 ОШИБКА: "Could not decode contract function call"

   ПРИЧИНА: Неправильный ABI или адрес контракта
   РЕШЕНИЕ:
   • Проверьте адреса токенов в NETWORKS
   • Убедитесь что токен существует в этой сети
   • Проверьте ABI соответствует контракту

   3. Обрабатывайте кошельки в дешевых сетях:
      • Polygon, BNB Chain, Avalanche - низкие комиссии


🔒 БЕЗОПАСНОСТЬ:

   1. НИКОГДА не делитесь приватными ключами
   2. Храните DONOR.txt в безопасном месте
   3. Используйте отдельный кошелек для RECIPIENT_ADDRESS
   4. Регулярно делайте backup файлов
   5. Проверяйте адреса перед запуском

📝 СЦЕНАРИЙ 1: БАЗОВЫЙ СБОР БЕЗ АВТОПОПОЛНЕНИЯ

   1. Настройка:
      RECIPIENT_ADDRESS = "0xYourAddress"
      RECIPIENT_ADDRESS_KEY = ""           # Оставить пустым
      AUTO_REFUEL_GAS = False
      MIN_BALANCE_USD = 0.05

   2. Убедитесь что на донорских кошельках есть газ

   3. Запуск:
      python token_collector.py

   4. Результат:
      • Собраны токены >= $0.05
      • LayerBank выведен (если есть)
      • SyncSwap выведен (если есть)
      • Все отправлено на RECIPIENT_ADDRESS


📝 СЦЕНАРИЙ 2: С АВТОПОПОЛНЕНИЕМ ГАЗА

   1. Настройка:
      RECIPIENT_ADDRESS = "0xYourAddress"
      RECIPIENT_ADDRESS_KEY = "0xYourPrivateKey"  # Ключ от адреса выше
      AUTO_REFUEL_GAS = True
      MIN_GAS_REFUEL_AMOUNT = 0.002

   2. Пополните RECIPIENT_ADDRESS (0.05-0.1 ETH на каждую активную сеть)

   3. Запуск:
      python token_collector.py

   4. Результат:
      • Если на кошельке нет газа → автопополнение
      • Все токены собраны
      • Остатки газа возвращены на RECIPIENT_ADDRESS


📝 СЦЕНАРИЙ 3: ТЕСТИРОВАНИЕ С 1 КОШЕЛЬКОМ

   1. Backup:
      copy DONOR.txt DONOR_backup.txt

   2. Оставьте 1 ключ в DONOR.txt

   3. Настройка:
      MAX_PARALLEL_WALLETS = 1
      PAUSE_BETWEEN_WALLETS = 0

   4. Запуск:
      python token_collector.py

   5. Проверка логов и результатов

   6. Восстановление:
      copy DONOR_backup.txt DONOR.txt

📊 АРХИТЕКТУРА:

   • Python 3.8+
   • Web3.py для взаимодействия с блокчейном
   • asyncio + ThreadPoolExecutor для многопоточности
   • Индивидуальные RPC для каждой сети (Ankr)


🔧 ОСНОВНЫЕ КОМПОНЕНТЫ:

   1. process_wallet_in_network()
      • Обрабатывает 1 кошелек в 1 сети
      • Проверяет LayerBank, SyncSwap, ERC-20, нативные токены

   2. check_and_refuel_gas()
      • Проверяет баланс газа
      • Пополняет автоматически если нужно

   3. refuel_gas_from_recipient()
      • Отправляет газ с RECIPIENT_ADDRESS
      • Поддерживает EIP-1559 и Legacy транзакции

   4. process_layerbank_withdrawals()
      • Выводит депозиты из LayerBank

   5. process_syncswap_withdrawals()
      • Выводит ликвидность из SyncSwap


🌐 RPC ENDPOINTS:

   Все сети используют Ankr RPC с API ключом:
   • Высокая скорость и надежность
   • Rate limits: ~100 req/sec
   • Резервные эндпоинты можно добавить в код

🆕 GNOSIS (Chain ID: 100)
   • Нативный токен: xDAI (~$1.00 стабильная цена)
   • RPC: Ankr Gnosis endpoint
   • Особенности:
     - xDAI = стейблкоин, привязан к USD
     - Низкие комиссии (~$0.01 за транзакцию)
     - Отличная скорость подтверждения
   
   Поддерживаемые токены:
   • EURA (0x4b1e2c2762667331bc91648052f646d1b0d35984)
   • LZ-agEUR (0xfa5ed56a203466cbbc2430a43c66b9d8723528e7)

🆕 CORE (Chain ID: 1116)
   • Нативный токен: CORE (~$1.20)
   • RPC: Ankr Core endpoint
   • Особенности:
     - Bitcoin-powered EVM blockchain
     - Hybrid consensus (PoW + DPoS)
     - Совместимость с Ethereum tooling
   
   Поддерживаемые токены:
   • USDT (0x900101d06a7426441ae63e9ab3b9b0f63be145f1)
   • BTC.b (0x2297aebd383787a160dd0d9f71508148769342e3)

🆕 ARBITRUM NOVA (Chain ID: 42170)
   • Нативный токен: ETH
   • RPC: Ankr Arbitrum Nova endpoint
   • Особенности:
     - Layer 2 решение на базе Arbitrum
     - Ultra-low fees (ультра-низкие комиссии)
     - Оптимизировано для gaming и social приложений
     - Использует AnyTrust технологию
     - Идеально для микротранзакций
   
   Поддерживаемые токены:
   • Токены будут добавлены по мере необходимости
   • Нативный ETH поддерживается полностью
